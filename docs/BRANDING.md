# Yapper Branding Guidelines

## App Name

**Yapper** - Voice Flows to Text

## Tagline

"AI-Powered Dictation for macOS"

## Brand Identity

### Mission
Yapper transforms spoken words into written text with AI-enhanced intelligence, making dictation natural, accurate, and context-aware for Mac users.

### Core Values
- **Privacy First**: Local processing by default, user owns their data
- **Intelligent**: AI-powered processing for better understanding
- **Seamless**: Works system-wide with any application
- **Fast**: Optimized for instant transcription with minimal latency

## Visual Identity

### Logo
See `docs/ICON_DESIGN.md` for the complete icon specification.

**Concept**: A waveform forming a "V" shape representing voice and Yapper.

### Colors

**Primary Palette:**
- **Flow Blue**: `#4A90E2` - Primary brand color, represents technology and trust
- **Voice Purple**: `#7C3AED` - Secondary color, represents AI and intelligence
- **Active Green**: `#10B981` - Accent for active/recording states

**Status Colors:**
- **Recording Red**: `#EF4444` - Recording in progress
- **Processing Blue**: `#3B82F6` - Transcribing/Processing
- **Success Green**: `#10B981` - Completed successfully
- **Warning Orange**: `#F59E0B` - Attention needed
- **Error Red**: `#DC2626` - Error state

**Neutral Colors:**
- **Gray 900**: `#111827` - Primary text
- **Gray 600**: `#4B5563` - Secondary text
- **Gray 400**: `#9CA3AF` - Placeholder text
- **Gray 200**: `#E5E7EB` - Borders
- **Gray 100**: `#F3F4F6` - Backgrounds

### Typography

**System Font**: SF Pro (macOS system font)
- **Headlines**: SF Pro Display, Bold/Semibold
- **Body**: SF Pro Text, Regular
- **Code/Monospace**: SF Mono

## Voice & Tone

### Writing Style
- **Clear and concise**: No jargon, simple explanations
- **Friendly but professional**: Helpful without being overly casual
- **Action-oriented**: Tell users what to do, not what not to do
- **Empowering**: Focus on what users can accomplish

### Example Copy

**Good:**
- "Press ⌘⌥Space to start recording"
- "Yapper needs microphone access to transcribe your voice"
- "Your recording will be transcribed locally on your Mac"

**Avoid:**
- "Don't forget to enable microphone access!"
- "You cannot use Yapper without..."
- "This feature is unavailable because..."

## User Interface Guidelines

### Iconography
- Use SF Symbols when possible for consistency
- Custom icons should follow SF Symbols style (rounded, consistent weight)
- Recording states should use clear, recognizable symbols

### Status Indicators
- **Idle**: Gray circle
- **Recording**: Red pulsing circle
- **Processing**: Blue animated spinner
- **Complete**: Green checkmark
- **Error**: Orange/red warning

### Window Styles
- **Menubar**: Minimal, icon-only or short text
- **Mini Window**: Compact, floating, translucent
- **Settings**: Clean, organized with clear sections
- **Recording Window**: Large, clear status with waveform
